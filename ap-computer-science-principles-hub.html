üìê AP CSP Big Ideas & Computational Thinking Practices

Big Idea 1: Creative Development (CRD)

Collaboration: Work with others to develop computing innovations.

Design process: Incorporate investigation, reflection, and iteration.

Program development: Design, implement, and evaluate programs.

Big Idea 2: Data (DAT)

Binary representation: Understand how data is stored and manipulated.

Data compression: Lossy vs. lossless techniques.

Data analysis: Extract information using computational tools.

Big Idea 3: Algorithms & Programming (AAP)

Variables & data types: Store and manipulate data.

Control structures: Sequencing, selection (if/else), iteration (loops).

Procedures: Define and call procedures with parameters.

Lists: Store and process collections of data.

Algorithms: Search, sort, and efficiency analysis.

Big Idea 4: Computing Systems & Networks (CSN)

The Internet: How data is transmitted; protocols (TCP/IP, HTTP).

Fault tolerance: How networks handle failures.

Parallel & distributed computing: Multiple processors working together.

Big Idea 5: Impact of Computing (IOC)

Beneficial & harmful effects: Analyze dual impacts of innovations.

Digital divide: Unequal access to technology.

Bias in computing: How algorithms can perpetuate bias.

Crowdsourcing: Leveraging collective intelligence.

Legal & ethical concerns: Privacy, intellectual property, security.

üî¢ Data & Binary

How Computers Store Information

Binary System

Everything is 0s and 1s (bits). 8 bits = 1 byte.

13 in binary: 1101 (8 + 4 + 0 + 1 = 13)

Data Representation
  • Numbers: Binary encoding
  • Text: ASCII/Unicode (A = 65 = 01000001)
  • Images: Pixels with RGB values
  • Sound: Samples of amplitude over time
Compression

Lossless: Original recoverable (ZIP, PNG)

Lossy: Some data lost (JPEG, MP3)

üåê The Internet

How Data Travels

Key Concepts
  • Packets: Data broken into small pieces
  • IP addresses: Unique device identifiers
  • Routing: Finding path through network
  • TCP/IP: Protocols ensuring delivery
Protocols

HTTP/HTTPS: Web page transfer (S = secure)

DNS: Translates domain names to IP addresses

Fault Tolerance

Multiple paths exist; if one fails, data takes another route.

Redundancy ensures reliability.

üîç Algorithms

Step-by-Step Problem Solving

Searching

Linear Search: Check each element (slow for large lists)

Binary Search: Divide and conquer on sorted list (much faster!)

Algorithm Efficiency

How does time grow as input size increases?

  • Linear: Doubling input doubles time
  • Quadratic: Doubling input quadruples time
  • Logarithmic: Very efficient; grows slowly

üìö Lesson 1: Introduction to Algorithms

‚è±Ô∏è 50 minutes

Learning Objectives

  • Students will define what an algorithm is
  • Students will write step-by-step instructions
  • Students will identify ambiguity in instructions

Lesson Procedure

Introduction (10 min): "Write instructions for making a peanut butter sandwich." Teacher follows instructions EXACTLY (often humorously wrong). Discuss why precision matters.

Activity (25 min): In pairs, write algorithm for a simple task. Partner tries to follow exactly. Iterate to fix problems.

Discussion (10 min): What makes a good algorithm? (Clear, unambiguous, complete, finite)

Closure (5 min): Algorithms are everywhere - recipes, directions, daily routines!

üìö Lesson 2: Binary Numbers

‚è±Ô∏è 55 minutes

Learning Objectives

  • Students will convert between decimal and binary
  • Students will understand why computers use binary
  • Students will represent simple data in binary

Lesson Procedure

Introduction (12 min): "Computers only understand ON and OFF - 1 and 0." Place values: instead of 10s, use 2s: 1, 2, 4, 8, 16, 32...

Guided Practice (18 min): Convert 13 to binary: 13 = 8+4+1 = 1101. Use binary cards activity.

Independent Practice (20 min): Convert numbers both directions; explore ASCII encoding.

Closure (5 min): "How would you represent 20 in binary?"

üìö Lesson 3: Variables and Conditionals

‚è±Ô∏è 55 minutes

Learning Objectives

  • Students will create and use variables
  • Students will write conditional statements
  • Students will trace program execution

Lesson Procedure

Introduction (12 min): Variables are like labeled boxes that store values. Demonstrate creating, assigning, and updating variables.

Guided Practice (18 min): Add conditionals: IF score >= 60 THEN pass ELSE fail. Trace through examples.

Independent Practice (20 min): Create programs using variables and conditionals.

Closure (5 min): "What value does x have after: x ‚Üê 5; x ‚Üê x + 3"

üìö Lesson 4: Lists and Loops

‚è±Ô∏è 55 minutes

Learning Objectives

  • Students will create and manipulate lists
  • Students will use loops to iterate through lists
  • Students will apply list algorithms

Lesson Procedure

Introduction (12 min): Lists store multiple values. Access by index (usually starting at 1 in AP CSP pseudocode). Operations: append, insert, remove.

Guided Practice (18 min): Loop through list to display each item, find sum, find maximum.

Independent Practice (20 min): Write programs that process lists.

Closure (5 min): "How would you find the average of a list?"

üìö Lesson 5: The Internet - How Data Travels

‚è±Ô∏è 50 minutes

Learning Objectives

  • Students will explain how data is transmitted across the Internet
  • Students will describe the purpose of protocols
  • Students will understand fault tolerance

Lesson Procedure

Introduction (10 min): "When you visit a website, what actually happens?" Trace the journey of a data packet.

Activity (25 min): Internet Simulator - students send messages through simulated network, observe routing and packet loss.

Discussion (10 min): Why break data into packets? What happens if one path fails?

Closure (5 min): The Internet's redundancy makes it resilient!

üìñ Key Terms

Algorithm

Definition: Step-by-step instructions to solve a problem

"A recipe is an algorithm for making food."

Abstraction

Definition: Simplifying complexity by focusing on essential features

"A map is an abstraction of the real world."

Binary

Definition: Base-2 number system using only 0 and 1

"13 in binary is 1101."

Protocol

Definition: Agreed-upon rules for communication

"HTTP is a protocol for web communication."

Iteration

Definition: Repeating a process; loops in programming

"A loop iterates through each item in a list."

Procedure

Definition: Named group of instructions (function)

"PROCEDURE calculateAverage(list) returns the mean."

‚ùì Common Misconceptions

‚ùå "The Internet and the World Wide Web are the same thing"

Correction: The Internet is the network infrastructure. The Web is one application that runs on it (along with email, video streaming, etc.).

‚ùå "More code is better code"

Correction: Efficient, readable code is better. Good programmers write less code that does more.
